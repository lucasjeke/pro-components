# @antdv-next1/pro-provider

Global configuration provider, internationalization (i18n), and theme context for Pro Components.

## Installation

```bash
npm install @antdv-next1/pro-provider
# or
pnpm add @antdv-next1/pro-provider
```

## Peer Dependencies

- `vue` >= 3.5.30
- `antdv-next` >= 1.3.6

## Usage

```vue
<script setup lang="ts">
import { ProConfigProvider } from '@antdv-next1/pro-provider'
</script>

<template>
  <ProConfigProvider :dark="false" :compact="false">
    <YourApp />
  </ProConfigProvider>
</template>
```

### Theme Integration

Use `ConfigProvider.theme` for antdv-next component tokens and `ProConfigProvider.token` for Pro component tokens.

```vue
<script setup lang="ts">
import { ref } from 'vue'

const dark = ref(false)
const compact = ref(false)
</script>

<template>
  <a-config-provider :theme="{ token: { colorPrimary: '#1677ff' } }">
    <ProConfigProvider
      :dark="dark"
      :compact="compact"
      :token="{
        layout: {
          header: {
            colorBgHeader: '#001529',
            colorHeaderTitle: '#fff',
          },
        },
      }"
    >
      <YourApp />
    </ProConfigProvider>
  </a-config-provider>
</template>
```

- `dark` and `compact` merge antdv-next algorithms into the current theme algorithm.
- Nested providers are scoped to their own subtree. Child provider tokens do not update unrelated global Pro components.
- Pro theme variables are emitted as CSS variables with the `css-var-pro` key, so dynamic theme changes update variable styles instead of reinserting component style text.
- Generated styles reuse `ConfigProvider.csp.nonce` when CSP is configured.

### Style Hook for Pro Components

`useStyle` is the shared CSS-in-JS helper for Pro packages. Component style hooks generated by `useStyle` return both the hash class and the CSS variable scope class:

```tsx
const [hashId, cssVarCls] = useStyle(prefixCls)

return () => (
  <div class={[prefixCls.value, hashId.value, cssVarCls.value]}>
    {slots.default?.()}
  </div>
)
```

When authoring a Pro component:

- attach both `hashId` and `cssVarCls` to the component root;
- keep component styles expressed with token values instead of hard-coded theme colors;
- pass `injectStyle: false` only when a component intentionally wants to skip runtime style injection;
- reset/common styles are registered before component styles so component selectors with overlapping prefixes keep priority.
